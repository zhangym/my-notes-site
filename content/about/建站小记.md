现在有了 AI 加持，搭建网站变得如此简单！

问 [豆包](https://www.doubao.com/chat/)：

> Cloudflare，Obsidian 如何部署自己的网站?

推荐两条路：

1. Obsidian 官方 Publish + Cloudflare 自定义域名（最简单、开箱即用，付费）
2. Obsidian 笔记 → 静态站点生成器（Quartz/VitePress/Hugo）→ GitHub → Cloudflare Pages（免费、可高度定制，稍复杂）
当然选第二种，问：

> Quartz, VitePress, Hugo 哪一个更好？

豆包给出：

1. 个人知识库 / 笔记（双向链接、关系图、Obsidian 联动）→ 选 Quartz 4
2. 技术文档 / 轻量博客、Vue 技术栈、追求极速热更新 → 选 VitePress
3. 大型博客 / 内容密集站、追求极致构建速度、成熟主题生态 → 选 Hugo
我选 [Quartz](https://github.com/jackyzha0/quartz)，不过最新更新是 20230821。暂且用着，待发现有更好的再去替换。

以下是我的建站记录：

1. 访问 [Quartz](https://github.com/jackyzha0/quartz)，
- fork 取名为 `InfoSphere`
- 在 `content` 下新建 `index.md`，随便写点内容。

```md
Hello Quartz!

第一次用`Cloudflare`部署Web站点！
```

3. 登录 [Cloudflare](https://dash.cloudflare.com)
- 注册 / 登录免费账号
 - 创建 Pages 项目：`Build/Compute/Workers & pages` → `create application`
	 - 在下面一行：`Looking to deploy Pages?`, 点击 `Get started`
	 - 选 `## Import an existing Git repository`, 点击 `Get started`
	 - `Select a repository` 选 `InfoSphere`
	 - `Production branch`: `v4`
	 - `Build command`: `npx quartz build`
	 - `Build output directory`: `public`, 然后 `Save and Deploy`
一路默认，自动部署。

3. 本地搭建 Quartz

```bash
# 1. 克隆 Quartz
git clone https://github.com/jackyzha0/quartz.git my-quartz
cd my-quartz

#! 由于网络原因，这一步git clone没有成功。我的做法是
# 从 https://github.com/jackyzha0/quartz 下载 quartz-4.zip
mkdir my-quartz
cd my-quartz


# 2. 安装依赖
npm install

# 3. 把你的 Obsidian 库复制到 quartz/content 下
# 比如：cp -r /path/to/你的Obsidian库/* content/

# 4. 本地预览
npx quartz build --serve
# 打开 http://localhost:8080 查看效果
```
